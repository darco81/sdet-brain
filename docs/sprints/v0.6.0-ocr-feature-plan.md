# v0.6.0 — OCR feature plan (image / PDF ingestion via DeepSeek-OCR + Qwen2.5-VL fallback)

**Status**: planned, ClickUp tasks created, ready to implement.

## TL;DR

Add image and PDF ingestion to sdet-brain. Receipts, invoices, whiteboard
photos, screenshots, multi-page PDFs flow through the same pipeline as
markdown — `parse_path(file)` dispatches by suffix to a new
`parse_image` / `parse_pdf` loader, runs OCR via a fallback chain
(MLX-VLM DeepSeek primary → Ollama DeepSeek secondary → Ollama
Qwen2.5-VL tertiary on Mac), returns markdown that hits the existing
chunker → embedder → Qdrant path. New MCP tool `ingest_image`.
Cross-platform sister release `0.2.0-win.0` follows on the Windows fork
with Ollama-only stack (4 GB VRAM constraint, no Qwen fallback).

## Fallback chain — Mac

```
1. mlx-vlm + mlx-community/DeepSeek-OCR-2-6bit  (~3s/img, primary)
   ↓ on health-fail / OCRQualityError / OCRTimeoutError
2. ollama + deepseek-ocr                        (~5-10s/img, secondary, same model different backend)
   ↓ on still-fail
3. ollama + qwen2.5-vl:32b                      (~40s/img, tertiary, heavyweight multilingual)
```

## Fallback chain — Win

```
1. ollama + deepseek-ocr  (~10-15s/img on RTX 3050 Ti)
   (no fallback — 4 GB VRAM cannot fit qwen2.5-vl:32b)
```

## Quality-fallback trigger heuristic

After primary succeeds, before returning:
- Output < `ocr_quality_min_chars` (default 50)
- Output contains only grounding tokens after strip (empty post-clean)
- Output is single line that's pure whitespace / punctuation

Raise `OCRQualityError`, factory catches and moves to next in chain.

## Edge cases covered (15)

| # | Edge case | Mitigation |
|---|---|---|
| 1 | EXIF rotation (phone photos) | `PIL.ImageOps.exif_transpose()` before OCR |
| 2 | HEIC from iPhone | `pillow-heif` registered globally on module import |
| 3 | PDF > 20 pages timeout | `OCR_MAX_PDF_PAGES=20` guard |
| 4 | Image > 20 MB | `OCR_MAX_IMAGE_BYTES=20_000_000` reject |
| 5 | Concurrent ingest race | `threading.Lock` in factory singleton |
| 6 | Idempotency after model change | content-hash on file bytes + payload `ocr_model` mismatch → force re-ingest |
| 7 | Privacy (PII in receipts) | Feature flag `OCR_PII_SCRUB=false` (off in MVP, hook ready) |
| 8 | Frontmatter auto-gen | Generated markdown: `source: ocr`, `ocr_model`, `extracted_at`, `original_path` |
| 9 | Watcher picks up image accidentally | Explicit watcher exclusion of image extensions |
| 10 | Memory pressure (Win 4 GB VRAM) | `OLLAMA_KEEP_ALIVE=5m` default — unload idle model |
| 11 | Polish OCR quality | DeepSeek-OCR multilingual, Qwen2.5-VL multilingual, both OK |
| 12 | Concurrent OCR contention | `semaphore=1` at factory level — protect VRAM |
| 13 | Logs flood | Structured INFO per call: `{duration_s, model, peak_gb}` |
| 14 | Search filter breakage | Existing payload index on `source_type` covers new enum value |
| 15 | MCP tool wrong-pick (Claude chooses `ingest_path` for image) | Tool description explicit about file types |
